# EntityDetailsPage

## Objetivo

Padronizar páginas e drawers de detalhes com Design System, abas, header, footer actions e estados de tela.

## Componentes usados

- `PageHeader`
- `AppCard`
- `DetailsSection`
- `InfoItem`
- `AppAlert`
- `AppCircularProgress`

## Estados

- loading
- error
- empty
- ready
- unauthorized
- forbidden

## Critérios de aceite

- Não conhece backend nem domínio.
- Pode ser usado por `platform` e `client`.
- Não renderiza erro técnico cru.
